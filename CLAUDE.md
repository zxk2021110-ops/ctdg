# 项目：宠物之家 (Pet Home)

宠物科普网站，运行在 Docker nginx 容器 `yyy` 中。

## 项目现状

已完成功能：
- 首页 (index.html) - 欢迎页，展示统计数据和导航入口
- 犬科图鉴 (dogs.html) - 展示犬种分类和详情，包含萨摩耶、雪橇犬系等
- 猫科图鉴 (cats.html) - 展示猫种分类，包含英短、布偶猫系等
- 饲养指南页 - 通用科普内容

现有设计风格参考截图：
- 配色：偏素净，需要优化为更温暖的宠物风格
- 布局：已实现卡片式展示，圆角可加强
- 导航：首页、犬科图鉴、猫科图鉴、饲养指南、退出登录

## 我的技术偏好

- 前端：原生 HTML + CSS + 原生 JavaScript（无框架）
- 部署：Docker + nginx 容器，容器名 yyy
- 配色风格：暖色调（米黄、浅橙、淡绿），柔和不刺眼
- 设计风格：圆角卡片、宠物友好、温暖可爱

## 优化方向

1. **视觉风格**：
   - 主色调改为暖色系（#FDF6E3 米黄背景，#FFB347 橙色点缀）
   - 所有按钮和卡片圆角加大至 20-30px
   - 增加爪印、宠物相关的装饰元素
   - 字体使用圆润风格

2. **功能增强**：
   - 搜索功能（搜索框已有前端，需实现过滤逻辑）
   - 首页增加“每日萌宠”或“随机科普小贴士”模块
   - 犬科/猫科图鉴页为每个品种增加“饲养难度星级”和“适合新手”标签
   - 饲养指南页使用折叠面板展示 FAQ

3. **响应式**：
   - 适配手机端（480px、768px 断点）
   - 移动端导航改为汉堡菜单

## 数据存储方案

采用纯前端JSON方案，不使用MySQL数据库。所有宠物数据存储在前端JavaScript对象或JSON文件中，搜索、过滤均在前端实现，保持架构简单。

若未来需要用户登录、评论、收藏等功能，再迁移到MySQL + 后端API架构。

## 输出要求

- 直接输出完整 HTML/CSS/JS 代码
- 保持现有页面间导航链接正确
- 新增功能使用原生 JS 实现，不引入外部库
